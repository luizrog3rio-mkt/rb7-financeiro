# Assinaturas — Introdução

> Referência: documentação Hotmart Developers — seção "Assinaturas" > "Introdução".

O serviço de assinaturas da Hotmart é um modo de **cobranças recorrentes**, programadas e processadas pelo **HotPay**.

## O que são Assinaturas

A modalidade implica **cobrança recorrente** pelo acesso ao conteúdo: o comprador faz pagamentos periódicos (mensal, bimestral, trimestral, semestral ou anual, conforme o produtor configurar) para manter o acesso ao produto.

- O Produtor define o **intervalo de tempo** entre as cobranças.
- Se não houver número de recorrências configurado, as cobranças seguem **até o comprador cancelar**.
- A data de cobrança é sempre o **mesmo dia** da aquisição (salvo alteração do assinante). Ex.: plano semestral iniciado em 01/01/2020 → próxima cobrança em 01/07/2020, e assim por diante.

## Status de uma assinatura

As assinaturas podem assumir diferentes status, que ajudam compradores, afiliados e produtores a acompanhar o momento do usuário:

| Status | Definição |
|---|---|
| **Ativa** | O pagamento da última recorrência está em dia e a assinatura ainda está no período de duração. |
| **Atrasada** | O cliente está com o pagamento da última recorrência atrasado, mas a assinatura ainda está no período vigente. |
| **Cancelada pelo Administrador** | Cancelamento feito pela Equipe de Suporte Hotmart. |
| **Cancelada pelo Cliente** | Cancelamento feito pelo comprador. |
| **Cancelada pelo Vendedor** | Cancelamento feito pelo Produtor ou seu colaborador. |
| **Inativa** | A primeira recorrência não teve pagamento confirmado/aprovado. |
| **Iniciada** | O aluno gerou o boleto da primeira recorrência, mas ainda não foi confirmado. |
| **Vencida** | Acabou o período de duração da assinatura. As recorrências do período podem estar todas quitadas ou haver alguma em atraso. |
